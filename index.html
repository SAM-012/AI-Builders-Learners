Bias–Variance Tradeoff

The bias–variance tradeoff explains why machine learning models make mistakes and how to balance them for better predictions.

  • Bias → Error from being too simple.
  • Variance → Error from being too sensitive to data.
  • Noise → Random error you can’t control.

Analogy: Archery

Bias: All arrows land in the same wrong corner → consistently wrong.

Variance: Arrows scatter everywhere → inconsistent.

Noise: Wind randomly moves arrows → unavoidable mistakes.

Goal: Arrows close together and near the bullseye → balanced model.

In Machine Learning

Bias (Underfitting): Model is too simple → misses important patterns.
Example: Predicting house prices using only the number of rooms.

Variance (Overfitting): Model is too complex → memorizes training data.
Example: Student memorizes past exam answers but fails new questions.

Noise: Unpredictable randomness in data.
Example: Identical houses sell for different prices due to negotiation.

The Tradeoff

Model Type Bias Variance Problem
Simple (Underfit) High Low Misses patterns
Complex (Overfit) Low High Memorizes noise
Balanced Medium Medium Generalizes well

Sweet Spot: Model complex enough to learn patterns but not so complex that it overfits.
